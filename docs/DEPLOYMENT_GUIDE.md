# Пошаговая инструкция по развертыванию GitHub Pages

## Шаг 1: Создание репозитория на GitHub

1. Перейдите на [GitHub](https://github.com)
2. Войдите в свой аккаунт (или создайте новый)
3. Нажмите кнопку **"+"** в правом верхнем углу → **"New repository"**
4. Укажите имя репозитория: `sportevents` (или любое другое)
5. Выберите **Public** (для GitHub Pages нужен публичный репозиторий в бесплатном плане)
6. Нажмите **"Create repository"**

## Шаг 2: Загрузка файлов

### Вариант A: Через веб-интерфейс GitHub (проще)

1. В созданном репозитории нажмите **"Add file"** → **"Upload files"**
2. Перетащите папку `docs` из вашего проекта
3. Или выберите файлы вручную:
   - `docs/index.html`
   - `docs/e/index.html`
   - `docs/i/index.html`
   - `docs/README.md`
4. В поле "Commit changes" напишите: "Add GitHub Pages files"
5. Нажмите **"Commit changes"**

### Вариант B: Через Git командную строку

```bash
# Перейдите в папку вашего проекта
cd C:\Users\megav\FlutterProj\sportEventsManager\sporteventmanager

# Инициализируйте Git (если еще не сделано)
git init

# Добавьте удаленный репозиторий
git remote add origin https://github.com/ВАШЕ_ИМЯ/sportevents.git

# Добавьте файлы
git add docs/

# Сделайте коммит
git commit -m "Add GitHub Pages files"

# Отправьте на GitHub
git push -u origin main
```

## Шаг 3: Включение GitHub Pages

1. В вашем репозитории перейдите в **Settings** (вкладка настроек)
2. В левом меню найдите **Pages**
3. В разделе **"Source"**:
   - Выберите ветку: **main**
   - Выберите папку: **/docs**
4. Нажмите **"Save"**
5. GitHub покажет сообщение: "Your site is ready to be published at..."
6. Скопируйте этот URL (например: `https://yourusername.github.io/sportevents/`)

## Шаг 4: Обновление ссылок в приложении

1. Откройте файл `lib/core/utils/event_link_utils.dart`
2. Найдите строку:
   ```dart
   const githubPagesUrl = 'https://yourusername.github.io/sportevents';
   ```
3. Замените на ваш реальный URL из Шага 3:
   ```dart
   const githubPagesUrl = 'https://ВАШЕ_ИМЯ.github.io/sportevents';
   ```

4. Откройте файл `lib/core/utils/institution_link_utils.dart`
5. Сделайте то же самое:
   ```dart
   const githubPagesUrl = 'https://ВАШЕ_ИМЯ.github.io/sportevents';
   ```

6. Сохраните файлы

## Шаг 5: Проверка

1. Подождите 2-5 минут после включения GitHub Pages
2. Откройте в браузере: `https://ВАШЕ_ИМЯ.github.io/sportevents/`
3. Вы должны увидеть главную страницу приложения
4. Проверьте редирект для мероприятий: `https://ВАШЕ_ИМЯ.github.io/sportevents/e/?id=test-123`
5. Должна открыться страница с попыткой открыть приложение

## Шаг 6: Тестирование в приложении

1. Создайте мероприятие в приложении
2. Нажмите кнопку "Поделиться"
3. Отправьте ссылку себе в Telegram/WhatsApp
4. Ссылка должна быть **кликабельной** и вести на вашу GitHub Pages
5. При клике должна открыться страница, которая попытается открыть приложение

## Возможные проблемы

### "404 Page Not Found"
- Подождите 5-10 минут после включения GitHub Pages
- Проверьте, что выбрана папка `/docs` в настройках Pages
- Убедитесь, что файлы загружены в папку `docs` в корне репозитория

### "Ссылки не работают"
- Проверьте, что вы обновили `githubPagesUrl` в обоих файлах утилит
- Убедитесь, что URL не содержит слэш в конце

### "Приложение не открывается"
- Deep links работают только если приложение установлено
- На iOS/Android нужно разрешить открытие приложений по ссылкам
- Проверьте, что настройки deep links в `AndroidManifest.xml` и `Info.plist` корректны
